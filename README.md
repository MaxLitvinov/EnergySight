
#  EnergySight — Моніторинг Енергетичних Об'єктів

> Практична робота №5 · Клієнтський веб-додаток (JavaScript + API)  
> Заняття №13–15 · Дисципліна: Веб-технології та веб-дизайн

---

##  Опис

Односторінковий веб-застосунок (SPA) для моніторингу енергетичних об'єктів у реальному часі. Реалізує взаємодію з REST API та WebSocket для завантаження і візуалізації телеметричних даних (потужність, температура, завантаженість агрегатів).

**Об'єкти моніторингу:**
- 🏭 ТЕС «Придніпровська» (900 МВт, 4 блоки)
- 💧 ГЕС «Дніпровська» (650 МВт, 3 агрегати)
- ☀️ СЕС «Херсонська» (200 МВт, 3 поля)
- 🌬️ ВЕС «Очаківська» (150 МВт, 4 ротори)

---

##  Демо

Відкрийте файл `energy_dashboard.html` у будь-якому сучасному браузері — встановлення сервера не потрібно.

```bash
# Клонувати репозиторій
git clone https://github.com/your-username/energy-sight.git
cd energy-sight

# Відкрити у браузері
open energy_dashboard.html       # macOS
start energy_dashboard.html      # Windows
xdg-open energy_dashboard.html   # Linux
```

##  Функціонал

| Компонент | Опис |
|-----------|------|
| **KPI-картки** | Потужність, енергія, завантаженість, температура з дельтою |
| **Лінійний графік** | Активна та реактивна потужність (30 останніх точок, оновлення 2 с) |
| **Кільцева діаграма** | Розподіл навантаження за типами споживачів |
| **Таблиця агрегатів** | Статус, потужність, температура кожного генератора |
| **Журнал подій** | Push-сповіщення в реальному часі (WebSocket) |
| **Перемикання об'єктів** | Миттєва зміна джерела даних |
| **Пауза / Відновлення** | Зупинка оновлень без втрати з'єднання |
| **Експорт CSV** | Завантаження даних графіка |

##  Технології

| Технологія | Призначення |
|------------|-------------|
| HTML5 | Структура застосунку |
| CSS3 (Custom Properties) | Темна тема, анімації, адаптивний макет |
| JavaScript ES2020 | Логіка, оновлення DOM |
| [Chart.js 4.4.1](https://www.chartjs.org/) | Лінійний графік та кільцева діаграма |

| Blob + URL API | Генерація CSV-файлу |

> **Залежності:** лише Chart.js через CDN. npm / Node.js не потрібні.

3. Випадковий інтервал 5–10 с — нові події у журналі

##  Підключення реального API

Для підключення до реального сервера замініть функцію `tick()`:

```js
// Замість симуляції:
const d = fakeRealtime(currentObj);

// Використовуйте реальний REST API:
const res = await fetch(`https://your-api.com/v1/objects/${currentObj}/realtime`);
const d = await res.json();

// Або WebSocket:
const ws = new WebSocket('wss://your-api.com/ws');
ws.onmessage = (event) => {
  const d = JSON.parse(event.data);
  updateKPIs(d);
  updateLineChart(d);
};
```
